Buttons and Button Contexts

Section 4.1

Buttons

Pattern Type:
atom

Button text should always be [chose a case] and no longer than [X] words long.

Display Examples:

Default styling
::hover
Hover state
.button-primary
When a button should be highlighted/is the main action in a set, use this class.
.button-primary::hover
When a button should be highlighted/is the main action in a set, use this class.

Markup Example:

<button class="[modifier class]">Button</button>
Section 4.2

Has-Buttons Contexts

Pattern Type:
molecule
Contains:
Only `buttons` or elements with the `.button` class

Like all context classes, a container with .has-buttons applied should only have elements that are buttons or have a .button class applied.

Display Examples:

Default styling
.right
right aligns the buttons inside the container
.stacked
stacks the buttons inside the container

Markup Example:

<div class="has-buttons [modifier class]">
  <button class="button-primary">Primary Button</button>
  <button>Default Button</button>
</div>
